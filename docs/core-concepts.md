# AI Agent 核心概念

---

## 1. 什么是 AI Agent？

**AI Agent** 是一个能够感知环境、做出决策并执行行动以实现目标的智能系统。

### Agent vs 传统程序

| 传统程序 | AI Agent |
|----------|----------|
| 固定逻辑 | 动态决策 |
| 明确输入输出 | 自主规划 |
| 无状态/简单状态 | 复杂记忆系统 |
| 被动响应 | 主动行动 |

## 2. 设计模式

### 2.1 ReAct (Reason + Act)

**核心思想**: 交替进行推理和行动

```
Thought: 我需要先搜索 X 的信息
Action: Search[X]
Observation: [搜索结果]
Thought: 根据搜索结果，我需要...
Action: ...
Final Answer: ...
```

**优点**:
- 可解释性强
- 能利用外部工具
- 适合复杂任务

**缺点**:
- 多次 LLM 调用，成本高
- 可能陷入循环

### 2.2 Plan-and-Solve

**核心思想**: 先制定完整计划，再逐步执行

```
1. 理解任务
2. 制定计划
3. 执行子任务
4. 汇总结果
```

**优点**:
- 全局视角
- 减少盲目行动

**缺点**:
- 计划可能不完善
- 灵活性较低

### 2.3 Reflection

**核心思想**: 执行后反思，持续改进

```
行动 → 观察 → 反思 → 调整 → 再行动
```

## 3. 记忆系统

### 短期记忆
- 对话历史
- 上下文窗口内
- 直接传递给 LLM

### 长期记忆
- 向量数据库存储
- 检索增强
- 跨会话保持

### 记忆类型对比

| 类型 | 实现方式 | 适用场景 |
|------|----------|----------|
| Buffer | 存储所有历史 | 短对话 |
| Summary | 自动摘要 | 长对话 |
| Vector | 向量检索 | 知识问答 |
| Entity | 实体提取 | 人物/事件追踪 |

## 4. 工具使用

### Tool 设计原则

1. **单一职责**: 每个工具做好一件事
2. **清晰描述**: 帮助 Agent 理解用途
3. **健壮性**: 处理错误和边界情况
4. **可组合**: 能与其他工具配合

### 常见工具类型

```python
# 搜索工具
SearchTool(query: str) -> str

# 计算工具
Calculator(expression: str) -> float

# API 工具
APITool(endpoint: str, params: dict) -> dict

# 文件工具
FileTool(operation: str, path: str) -> str
```

## 6. 评估指标

| 指标 | 说明 | 目标 |
|------|------|------|
| 任务完成率 | 成功完成的任务比例 | >90% |
| 工具准确率 | 正确调用工具的比例 | >95% |
| 响应时间 | 平均响应延迟 | <5s |
| Token 效率 | 每任务的 Token 消耗 | 优化 |
| 用户满意度 | 主观评价 | 4.5+/5 |

## 7. 常见挑战

### 7.1 幻觉问题
**现象**: Agent 生成虚假信息  
**解决**: RAG、工具验证、引用来源

### 7.2 无限循环
**现象**: Agent 重复相同行动  
**解决**: 循环检测、最大迭代次数

### 7.3 上下文溢出
**现象**: 超出 Token 限制  
**解决**: 摘要、截断、向量检索

### 7.4 工具调用错误
**现象**: 参数错误或工具选择错误  
**解决**: 更好的描述、Few-shot 示例
